## 2. 숫자·금액 표기

### 한국어 단위 우선
- 10,000 이상: 만 단위 사용 → `1만원`, `500만원`
- 100,000,000 이상: 억 단위 사용 → `1억원`, `3억 5,000만원`
- 혼용 가능: `1억 2,500만원` (두 단위 동시 표기)

### 입력 필드 인디케이션
- 금액 입력란 우측 하단에 실시간으로 한국어 단위 표시
- 구현: `data-amount-hint="원"` 또는 `data-amount-hint="만원"` 속성
- 예: `30,000` 입력 시 → `3억원` 표시 (단위가 만원인 경우)

### 결과값 표기 규칙
- 1만원 미만: `9,500원`
- 1만 ~ 1억 미만: `450만 3,200원`
- 1억 이상: `1억 2,300만원`
- 소수점: 금액은 원 단위 반올림, 이율은 소수점 2자리까지

## 3. 버튼 레이블

| 상황 | 나쁜 예 | 좋은 예 |
|------|---------|---------|
| 계산 실행 | 확인 / 조회 | 계산하기 |
| 초기화 | 리셋 / 취소 | 다시 입력 |
| 복사 | 클립보드에 복사 | 복사 |
| 복사 완료 | - | 복사됨 ✓ (0.8초 후 원복) |

**규칙:**
- 버튼은 동사로 시작 (계산하기, 확인하다 X → 확인 O)
- 주 액션(계산하기): Primary 색상, 전체 너비 또는 큰 비중
- 보조 액션(다시 입력): 회색 계열, 계산하기보다 시각적 비중 낮게

## 4. 입력 필드

### 레이블
- 사용자 관점으로 작성: `대출 원금 (만원)` → `빌리는 금액 (만원)`
- 단위는 레이블에 괄호로 표기: `연봉 (만원)`, `이자율 (%)`
- 필수 항목 표시: 별표(\*) 대신 레이블 뒤 회색 텍스트로 `(필수)` 또는 선택항목에만 `(선택)`

### 플레이스홀더
- 입력 예시만 표기, 설명 X: `예: 30,000` (설명 포함 금지: `만원 단위로 입력하세요`)
- 단위가 명확하면 단위 생략: `예: 4.5` (이율 필드)
- 없어도 되는 경우 플레이스홀더 미사용 (레이블만으로 충분하면)

### 도움말 텍스트 (hint)
- 필드 하단 회색 소형 텍스트로 보충 설명
- 금액 인디케이터: 파란색으로 실시간 한국어 금액 표시
- 너무 많으면 인지 부하 증가 — 꼭 필요한 필드에만

## 5. 에러 메시지

| 상황 | 나쁜 예 | 좋은 예 |
|------|---------|---------|
| 필수 입력 누락 | 오류 발생 | 대출금액을 입력해주세요 |
| 범위 초과 | 잘못된 값 | 이자율은 0~100% 사이로 입력해주세요 |
| 숫자 아닌 값 | 형식 오류 | 숫자만 입력해주세요 |

**규칙:**
- 무엇이 잘못됐는지 + 어떻게 고치면 되는지 함께 알려줌 (Apple HIG)
- `alert()` 사용 금지 — 인라인 에러 메시지 또는 결과 박스 위에 표시
- 색상: `var(--danger-500)` (#EF4444), 아이콘(⚠) 병행 가능

## 6. 결과 표시

### 계층 구조 (토스 카드 디자인 참고)
1. **핵심 결과** 1개: 크게, 굵게, Primary 색상 (예: 월 상환액 125만원)
2. **보조 결과** 2~3개: 중간 크기 (예: 총 이자, 총 상환액)
3. **상세 내역**: 접기/펼치기 또는 표로 하단 배치

### 레이블 → 값 쌍 작성법
- 레이블: 명사형, 간결 (`총 납부 이자`)
- 값: 단위 포함, 오른쪽 정렬 (`1,234만 5,600원`)
- 강조 결과는 `.result-value.highlight` 클래스로 구분

### 빈 상태
- 계산 전: 결과 박스 숨김 (display: none)
- "계산하기" 클릭 전까지 결과 영역 노출 금지 (기존 정책 유지)

## 7. 섹션 제목 & 설명문

### 페이지 헤더
- H2: 도구 이름 (간결) — `대출이자 계산기`
- 부제: 한 줄 설명 — `원리금균등·원금균등·만기일시 상환을 비교해보세요`
- 부제는 기능 나열보다 사용자 관점: `이자 계산하기` X → `이 정도 이자면 감당할 수 있을까요?` (토스 스타일)

### 섹션 구분
- 폼과 결과 사이 `<hr>` 또는 여백으로만 구분 (불필요한 소제목 금지)
- 관련 도구 섹션: `이런 계산도 해보세요` (네이버 스타일)

## 8. 모바일 UX 특이사항

- 숫자 입력: `inputmode="numeric"` 속성으로 모바일 숫자 키패드 강제 (현재 미적용 — 추가 권장)
- 탭/토글: 손가락 탭 영역 최소 44×44px (Apple HIG 기준)
- 스크롤 후 결과: `scrollIntoView({ behavior: 'smooth' })` — 현재 구현됨
- 앵커 광고(하단 고정) 고려: 결과 버튼이 광고에 가려지지 않도록 하단 padding 확보

## 9. 어조 (Tone of Voice)

| 상황 | 어조 | 예시 |
|------|------|------|
| 일반 안내 | 중립·정중 | "입력하세요", "확인하세요" |
| 결과 긍정 | 따뜻·공감 | "예상 실수령액이에요" (토스 스타일 — 선택적 적용) |
| 에러 | 차분·도움 | "다시 확인해주세요" (책망 금지) |
| 주의 정보 | 명확 | "실제 금액과 다를 수 있어요" |

현재 사이트 어조: **중립·정중** 유지 (신뢰감 우선, 지나친 친근감 지양)

## 10. 즉시 적용 가능한 개선 항목 (우선순위순)

| 우선순위 | 항목 | 설명 |
|---------|------|------|
| ★★★ | 금액 인디케이터 | 입력 중 한국어 단위 실시간 표시 — **이미 구현됨** |
| ★★★ | `inputmode="numeric"` | 모바일 숫자 키패드 — 금액 입력 필드에 추가 |
| ★★☆ | 에러 인라인화 | alert() 없이 필드 아래 에러 표시 |
| ★★☆ | 결과 레이블 사용자화 | "계산 결과" → 도구별 맞춤 문구 |
| ★☆☆ | 관련 도구 문구 | "관련 도구" → "이런 계산도 해보세요" |
