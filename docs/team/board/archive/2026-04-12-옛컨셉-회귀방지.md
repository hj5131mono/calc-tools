- **배경**: 사용자가 본 세션에서 직접 발견한 결정적 결함 — 도구 페이지(주휴수당 등) 상단바가 폐기된 옛 카테고리(홈/금융투자/노무급여/로또운세/일상 유틸리티) hardcoded 헤더로 표시. common.js initLayout() 강제 교체 로직으로 1차 해결됐지만 SEO·코드 일관성·유지보수 측면에서 hardcoded 자체를 제거하는 본질 정리가 필요.
- **범위**:
  - 도구 페이지 13개의 hardcoded `<header class="site-header">...</header>` + `<footer class="site-footer">...</footer>` 블록 제거
  - 영향 파일: `finance/loan, compound, salary, vat, percent` (5) / `daily/age, dday, bmi, character-count` (4) / `stock/return-rate, averaging` (2) / `lotto/saju-pick` (1) / `privacy.html` (1)
  - `robots.txt` 옛 카피 ("계산기 도구 모음" → "calc-tools.kr")
  - `assets/og-image.svg` 옛 텍스트 → 백로그 등록 (이미지 편집 별도)
  - 회귀 방지 자동 검사 (pre-commit hook 또는 GitHub Actions check)
- **성공 기준**:
  - 13개 파일에서 hardcoded 헤더/푸터 완전 제거 + common.js initLayout() 자동 생성으로 정상 작동
  - 회귀 방지 검사가 옛 키워드 11개 이상 catch
  - 모든 변경 후 13개 페이지 sample 점검 시 새 헤더 (홈/핵심 도구/가이드/소개/문의) + "calc-tools.kr" 로고 정상
- **우선순위**: P0
- **첫 미션이라는 의미**: 본 미션은 셋업 직후 첫 미션이며, MD board 운영 모델의 검증·튜닝 케이스이기도 함.

## 핵심 의사결정 (PM)
1. **회귀 방지 검사 위치**: pre-commit hook (husky 도입) vs GitHub Actions check vs 단순 bash script
   → **결정: 단순 bash + GHA 둘 다**. 이유: husky는 npm 의존성 추가 부담, 단순 bash는 로컬 실행 용이, GHA는 강제성 보장.
2. **og-image.svg 처리**: 본 미션 범위 vs 백로그
   → **결정: 백로그 등록**. 이유: SVG 이미지 편집은 텍스트 편집보다 복잡, 디자인 의사결정 (Brand 단독 작업)이 적합.
3. **privacy.html 포함 여부**: 도구 페이지가 아니지만 hardcoded 헤더 보유
   → **결정: 포함**. 일관성 우선.
