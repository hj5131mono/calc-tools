# PM 누적 노트

> PM이 매 미션에서 발견한 패턴·인사이트·차별화 노하우를 누적하는 노트.
> 호출 시 작업 시작 전에 본 파일을 먼저 읽고 이전 학습을 회복한다.

## 차별화 5축 (calc-tools 정체성)
1. **한국 특화도 100%** — 글로벌 계산기가 못 만드는 한국 노동법·세법 도구
2. **법령 1차 출처** — 법제처 직링크 (E-E-A-T 신호)
3. **교육 + 도구 결합** — 계산만 하지 않고 왜·어떻게·얼마인지 끝까지 설명
4. **Grey Zone 영리 활용** — 사용자가 받을 수 있는 합법적 최대 효용 적극 발굴
5. **단일 진실 공급원 (rates.json)** — 매년 변경 수치도 한 곳만 갱신

## 패턴 라이브러리

### 미션 분배 결정 트리
- 가이드 글 작성 → Compliance(본문) → Brand(검수) → DevOps(sitemap) → QA(링크)
- 산식 검증 → QA → Compliance → DevOps → 보고
- rates 갱신 → DevOps → Compliance → QA
- UI 변경 → Brand → DevOps → QA
- AdSense 점검 → 전원

### 인사이트
- 가이드 글 템플릿이 한 번 정립되면 작성 속도 4~5배 (세션 39)
- 글 1편당 800~1500단어가 적정 (한 세션 5~7편)
- 산식 검증 패스는 작성 직후가 아니라 글 모두 완성 후 별도로 (산식 오류는 1번 패스로 못 잡음)
