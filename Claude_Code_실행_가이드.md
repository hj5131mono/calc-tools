# 자동사냥 MVP 구축 — Claude Code 실행 가이드

> **이 문서의 사용법:** 위에서부터 순서대로 따라하면 된다.

## Phase 0: 사전 준비 (PC에 도구 설치하기)

### 0-1. Node.js 설치

Claude Code는 Node.js 위에서 돌아간다. 없으면 설치해야 한다.

1. https://nodejs.org 접속
2. **LTS 버전** (초록색 버튼) 다운로드 후 설치
3. 설치 확인: 터미널(또는 명령 프롬프트)을 열고 아래 입력

```bash
node --version
```

`v18.x.x` 이상이 나오면 OK.

이미 깔려 있으면 건너뛰기.

1. https://code.visualstudio.com 에서 다운로드 후 설치

GitHub에 코드를 올려야 무료 호스팅을 쓸 수 있다.

1. https://git-scm.com 에서 다운로드 후 설치
2. 설치 확인:

```bash
git --version
```

## Phase 1: 프로젝트 생성

### 1-1. 프로젝트 폴더 만들기

원하는 위치에 폴더를 만든다. 터미널에서:

```bash
code .
```

또는 VS Code → File → Open Folder → 방금 만든 `calc-tools` 폴더 선택

### 1-3. CLAUDE.md 파일 만들기 (매우 중요!)

프로젝트 루트에 `CLAUDE.md` 파일을 만든다.  
이 파일은 Claude Code가 프로젝트를 이해하는 컨텍스트 역할을 한다.  
VS Code에서 새 파일 → `CLAUDE.md` → 아래 내용을 그대로 복사 붙여넣기:

```markdown
# 프로젝트: calc-tools (자동사냥 유틸리티 사이트)

## 프로젝트 목적
검색 유입 → 도구 사용 → 애드센스 광고 수익을 자동으로 발생시키는 정적 유틸리티 사이트.
배포 후 유지보수 0을 목표로 한다.

## 기술 스택
- 순수 HTML + CSS + Vanilla JavaScript (프레임워크 없음)
- 빌드 과정 없음 (HTML 파일을 그대로 서빙)
- 서버 없음 (모든 계산은 클라이언트 JS로 처리)
- 호스팅: GitHub Pages 또는 Cloudflare Pages (무료)

## 디자인 원칙
- 모바일 퍼스트 반응형 디자인 (트래픽의 70%+ 가 모바일)
- 깔끔하고 신뢰감 있는 UI (광고 클릭률에 직접 영향)
- 계산 결과 영역 위/아래에 광고 영역 확보 (div class="ad-space")
- 페이지 로딩 속도 최우선 (Core Web Vitals 통과)
- 한국어 전용 사이트

## 파일 구조 규칙
- 모든 HTML 파일은 독립적으로 동작 (공통 CSS/JS는 인라인 또는 공유 파일)
- /assets/css/style.css → 공통 스타일시트
- /assets/js/common.js → 공통 함수 (숫자 포맷, 내비게이션 등)
- 각 도구 페이지는 카테고리 폴더 아래에 위치
- 예: /stock/averaging.html, /finance/loan.html

## SEO 규칙
- 모든 페이지에 고유한 <title>과 <meta name="description"> 필수
- Open Graph 태그 포함
- 한국어 lang="ko" 설정
- sitemap.xml과 robots.txt 루트에 배치
- 내부 링크를 가능한 많이 연결 (관련 도구끼리)

## 광고 영역 규칙
- 각 페이지 계산 결과 위/아래에 아래 마크업 삽입:
  <div class="ad-space" style="min-height:90px; text-align:center; margin:20px 0;">
    <!-- AdSense 광고 자리 -->
  </div>
- 페이지당 광고 영역 2~3개
- 실제 애드센스 코드는 나중에 승인 후 삽입

## 코딩 규칙
- const/let 사용 (var 금지)
- 주석은 한국어로 작성
- 숫자 출력 시 천단위 콤마 적용 (toLocaleString 사용)
- input에 적절한 placeholder와 label 제공
- 계산 버튼과 초기화 버튼 모두 제공
```

저장한다. (`Ctrl+S`)

```
1. /assets/css/style.css — 공통 스타일시트를 만들어줘:
   - CSS 리셋 (margin, padding, box-sizing)
   - 모바일 퍼스트 반응형 레이아웃 (max-width: 720px, 가운데 정렬)
   - 색상 변수: --primary: #2563eb, --bg: #f8fafc, --text: #1e293b, --border: #e2e8f0
   - 공통 컴포넌트: .calc-card (흰 배경, 라운드, 그림자), .btn-primary (파란 버튼), .btn-reset (회색 버튼), input/select 스타일, .result-box (결과 표시 영역, 배경 하이라이트), .ad-space (광고 영역 placeholder)
   - 반응형: 768px 이상일 때 레이아웃 조정
   - 네비게이션: .site-header, .nav-links
   - 푸터: .site-footer

2. /assets/js/common.js — 공통 유틸리티:
   - formatNumber(num): 천단위 콤마 포맷
   - parseNumber(str): 콤마 제거 후 숫자 변환
   - 입력 필드에 숫자 입력 시 자동 콤마 포맷팅 함수
   - 공통 네비게이션 HTML을 동적으로 삽입하는 함수 (사이트 이름: "계산기 도구 모음")

3. index.html — 메인 허브 페이지:
   - 사이트 제목: "계산기 도구 모음 — 일상에 필요한 모든 계산기"
   - 카테고리별로 도구 카드 목록 표시 (링크 포함)
   - 카테고리: 금융/투자 계산기, 로또/운세, 일상 유틸리티
   - 각 도구 카드에 아이콘(이모지 OK) + 이름 + 한줄 설명 + 링크
   - SEO 메타태그, Open Graph 태그 포함
   - 광고 영역 2개 배치 (상단, 하단)

4. privacy.html — 개인정보처리방침 페이지 (애드센스 필수)
5. robots.txt
6. sitemap.xml (일단 빈 구조, 페이지 추가될 때마다 업데이트)

CLAUDE.md에 정의된 규칙을 모두 따라줘.
```

→ Claude가 파일들을 생성하면 diff를 확인하고 **Accept** 해준다.

### 2-7. 핵심 도구 #5 — 사주 행운번호 추천기 (바이럴 킬러)

📋 **Claude Code에 입력:**

```
/lotto/saju-pick.html 사주 기반 로또 행운번호 추천기를 만들어줘.

## 페이지 컨셉
사용자의 생년월일시, 성별, MBTI를 입력받아 사주팔자(천간지지)를 계산하고,
오행 분석 결과를 기반으로 "나만의 행운 번호"를 추천하는 재미형 도구.
반드시 면책조항: "본 서비스는 사주 명리학을 활용한 재미/참고용이며, 당첨을 보장하지 않습니다."

## 입력 UI
1. 생년월일: date input (양력 기준)
2. 태어난 시간: select로 12시진(時辰) 선택
   - 자시(23:00~01:00), 축시(01:00~03:00), 인시(03:00~05:00),
     묘시(05:00~07:00), 진시(07:00~09:00), 사시(09:00~11:00),
     오시(11:00~13:00), 미시(13:00~15:00), 신시(15:00~17:00),
     유시(17:00~19:00), 술시(19:00~21:00), 해시(21:00~23:00),
     "모름" 옵션 (모름 선택 시 시주 제외하고 년월일주만 사용)
3. 성별: 남/여 라디오버튼
4. MBTI: 4개 축 각각 선택 (E/I, S/N, T/F, J/P) — select 또는 버튼 토글

## 사주 계산 로직 (JavaScript로 구현)

### 기본 데이터 상수
천간 10개: 갑,을,병,정,무,기,경,신,임,계
지지 12개: 자,축,인,묘,진,사,오,미,신,유,술,해
천간오행: 갑을=목, 병정=화, 무기=토, 경신=금, 임계=수
지지오행: 자=수, 축=토, 인묘=목, 진=토, 사오=화, 미=토, 신유=금, 술=토, 해=수
오행 색상: 목=#22c55e, 화=#ef4444, 토=#eab308, 금=#d1d5db, 수=#3b82f6
오행 → 로또 번호대 매핑:
  목: 1~9, 화: 10~18, 토: 19~27, 금: 28~36, 수: 37~45

### 년주(年柱) 계산
간 = ((year - 4) % 10 + 10) % 10 → 천간 인덱스
지 = ((year - 4) % 12 + 12) % 12 → 지지 인덱스

### 월주(月柱) 계산
월간 = (년간인덱스 * 2 + month) % 10
월지 = (month + 1) % 12  (1월=인, 2월=묘 ...)

### 일주(日柱) 계산 — 율리우스 적일(JDN) 기반
JDN 공식으로 날짜를 일련번호로 변환 후:
기준일: 2000-01-07 = 갑자일 (JDN 2451551)
offset = ((JDN - 2451551) % 60 + 60) % 60
일간 = offset % 10, 일지 = offset % 12

### 시주(時柱) 계산
시지 = 선택한 시진 인덱스 (자=0, 축=1 ...)
시간 = (일간인덱스 * 2 + 시지인덱스) % 10

### MBTI → 오행 보정
E=화, I=수, S=토, N=목, T=금, F=화, J=금, P=수
```

### 6-2. 호스팅KR에서 DNS 설정

**접근 경로:** 로그인 → 우측 상단 **나의 서비스** → **도메인** → 구매한 도메인 클릭 → **네임서버/DNS** 탭

DNS 레코드 관리 화면에서 **"새 DNS 레코드 추가"** 클릭.

#### A 레코드 4개 추가 (루트 도메인 연결)

아래 4개를 **각각 따로** 추가한다:

| 유형 | 호스트 | 값 |
|------|--------|-----|
| A | @ | 185.199.108.153 |
| A | @ | 185.199.109.153 |
| A | @ | 185.199.110.153 |
| A | @ | 185.199.111.153 |

> `@`는 루트 도메인(예: calc-tools.kr)을 의미한다.

#### CNAME 레코드 1개 추가 (www 연결)

| 유형 | 호스트 | 값 |
|------|--------|-----|
| CNAME | www | 너의유저네임.github.io |

> `너의유저네임`은 실제 GitHub 아이디로 바꿔야 한다.

**DNS 반영 시간:** 빠르면 5분, 최대 48시간 소요. 보통 1~2시간이면 적용된다.

### 6-3. GitHub Pages에서 도메인 설정

1. GitHub 저장소 → **Settings** 탭 → 왼쪽 **Pages** 메뉴
2. **Custom domain** 입력란에 구매한 도메인 입력 (예: `calc-tools.kr`)
3. **Save** 클릭
   - 이 때 저장소 루트에 `CNAME` 파일이 자동 생성된다 (도메인명이 저장된 파일)
4. DNS 반영 확인 후 **"Enforce HTTPS"** 체크박스 활성화
   - DNS가 아직 전파되지 않은 상태면 이 옵션이 비활성화되어 있다. 1~2시간 후 다시 시도.

> **CNAME 파일 주의:** GitHub가 자동 생성한 `CNAME` 파일을 절대 삭제하지 말 것.
> 다음 `git pull`을 먼저 실행해서 이 파일을 로컬에도 받아와야 한다.

```bash
git pull origin main
```
